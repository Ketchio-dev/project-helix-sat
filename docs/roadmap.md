# Execution Roadmap

## Phase 0 — Foundation
- Freeze v1 skill ontology
- Freeze error taxonomy
- Freeze canonical rationale + hint schemas
- Define starter data model and API boundaries
- Stand up verification scripts

## Phase 1 — Learning Core MVP
- Onboarding
- Diagnostic session
- Attempt/event ingestion
- Learner-state update loop
- Daily plan generation
- Drill + review loop
- Tutor hint flow with canonical grounding

## Phase 2 — Test Core
- Timed sets
- Module simulation
- Projected score bands
- Pacing analytics
- Exam-mode hardening

## Phase 3 — Content Operations
- CMS/editor
- Item review workflow
- Field-test/calibration ingest
- Defect triage dashboard
- Content release discipline

## Phase 4 — Expansion
- Parent reports
- Teacher dashboard
- Org seats and assignments
- Score-delta planner upgrades
- Error DNA visualization

## Current product-completion focus

Before broadening Phase 4 surfaces, prioritize learner-product completion:
- first-15-minute activation
- post-session conviction
- review-as-remediation
- visible personalization
- SAT fidelity and content depth
- retention through automatic next actions

See `docs/product-completion-milestones.md` for the concrete milestone and issue tree.
